============================================================
  测试用例记录表 — 使用说明
============================================================

【 这是什么 】
  将 Excel 中的测试用例逐条展示为清晰易读的 HTML 卡片页面，
  执行后记录测试结果和备注，自动回写 Excel。

【 怎么用 】
  1. 把你的测试用例 Excel 文件（.xlsx 或 .xls）放到 testcases 文件夹里
  2. Mac：双击 start.command
     Windows：双击 start.bat
  3. 程序会自动安装依赖（仅首次需要）、打开浏览器
  4. 在网页中逐条执行测试、记录结果
  5. 结果会自动写回 Excel

【 Excel 格式要求 】
  - 第一行为表头（列名）
  - 从第二行开始为用例数据
  - 程序会自动识别常见列名：
    编号、标题、前置条件、测试步骤、预期结果、
    测试目的、优先级、模块、分类 等
  - 未识别的列名也会展示

【 功能说明 】
  三个视图（顶部标签页切换）：

  ① 执行测试 — 逐条展示用例卡片，选择结果、填写备注、保存
  ② 搜索筛选 — 按关键词搜索，按结果/优先级/模块筛选，点击跳转
  ③ 汇总统计 — 查看执行进度、通过率、按优先级/模块分布

【 键盘快捷键 】
  ← → 方向键：切换上一条/下一条
  Ctrl+S：保存当前结果

【 系统要求 】
  - Python 3.8+
  - Mac 默认自带 Python 3
  - Windows 需先安装 Python：https://www.python.org/downloads/
    （安装时勾选 "Add Python to PATH"）

【 文件说明 】
  start.command  — Mac 启动脚本
  start.bat      — Windows 启动脚本
  testcase_viewer.py — 主程序
  requirements.txt   — Python 依赖清单
  testcases/         — 放置测试用例 Excel 的目录（第一个 .xlsx 文件会被读取）

【 常见问题 】
  Q: 保存时提示"无法写入 Excel 文件"
  A: 请关闭正在打开的 Excel 程序后重试。

  Q: 浏览器没有自动打开
  A: 手动访问 http://127.0.0.1:8765

  Q: 需要切换 Excel 文件
  A: 把新 Excel 放入 testcases/，删除或移走旧文件，重新启动程序。
