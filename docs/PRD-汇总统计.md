## 一、需求背景

- **问题**：当前测试用例 Excel 通常包含多个 sheet，部分是用例 sheet，部分是测试备注、场景说明、测试结果总览等非用例内容。当前工具只识别一个 sheet，无法区分和展示其他 sheet 的内容。
- **价值**：用户需要一眼看完所有用例 sheet 的测试结果分布，方便后期回归 bug 和选择优先测试哪个模块。
- **目标用户**：测试执行人员、测试管理者
- **使用场景**：
  1. 测试执行前，快速浏览各 sheet（模块）的执行进度，选择优先测试的模块
  2. 回归测试时，查看哪些 sheet 的未执行/失败用例最多，优先处理

## 二、功能描述

- **一句话描述**：自动识别 Excel 中所有 sheet 的用例/非用例类型，将测试用例按 sheet 分组汇总展示，并单独展示非用例 sheet 的内容。
- **输入**：用户上传的 Excel 文件（.xlsx/.xls），可能包含多个 sheet
- **输出**：
  1. 按 sheet 分组的用例汇总列表（用例名称、序号、结果）
  2. 非测试用例 sheet 的原始内容展示
- **触发方式**：页面加载时自动解析，无需手动触发

## 三、实现方案

### 3.1 技术栈

- 后端：Python Flask + openpyxl（已存在）
- 前端：原生 HTML/CSS/JS（已存在）
- 无新增依赖

### 3.2 Sheet 分类引擎

新增 `classify_sheets(filepath)` 函数，对每个 sheet 判断类型：

```python
def classify_sheet(sheet_name, ws):
    """
    判断一个 sheet 是否为测试用例 sheet。
    
    判断标准（复用现有 _pick_best_sheet 逻辑）：
      1. Sheet 名称包含"用例/Case/TestCase"等关键词的优先
      2. 必须具备 >= 2 个核心用例字段（id, title, steps, expected, precondition, priority, module, category）
      3. 有至少 1 条数据行
    
    返回：('testcase', header_row, mapping, data_rows) 或 ('non_testcase', reason)
    """
```

**核心字段定义**：

| 字段名 | 说明 |
|--------|------|
| id | 用例编号/序号 |
| title | 用例标题/名称 |
| steps | 测试步骤 |
| expected | 预期结果 |
| precondition | 前置条件 |
| priority | 优先级 |
| module | 所属模块 |
| category | 用例分类 |

### 3.3 后端 API

| 接口 | 方法 | 路径 | 作用 |
|------|------|------|------|
| sheets-overview | GET | `/api/sheets-overview` | 返回所有 sheet 的分类信息（用例/非用例） |
| sheet-summary | GET | `/api/sheet-summary` | 返回按 sheet 分组的用例汇总数据 |
| sheet-meta | GET | `/api/sheet-meta/<sheet_name>` | 返回非用例 sheet 的原始内容（表格形式） |

**API 1：`/api/sheets-overview` 返回格式**

```json
{
  "sheets": [
    { "name": "登录用例", "type": "testcase", "count": 25 },
    { "name": "支付用例", "type": "testcase", "count": 30 },
    { "name": "测试备注", "type": "non_testcase", "reason": "无核心用例字段" },
    { "name": "场景说明", "type": "non_testcase", "reason": "无核心用例字段" }
  ],
  "testcase_sheet_names": ["登录用例", "支付用例"],
  "non_testcase_sheet_names": ["测试备注", "场景说明"],
  "total_testcases": 55
}
```

**API 2：`/api/sheet-summary` 返回格式**

```json
{
  "groups": [
    {
      "sheet_name": "登录用例",
      "count": 25,
      "counts": { "通过": 10, "失败": 5, "阻塞": 2, "未执行": 8 },
      "testcases": [
        { "global_index": 0, "sheet_index": 0, "id": "TC-001", "title": "正常登录", "result": "通过" },
        { "global_index": 1, "sheet_index": 1, "id": "TC-002", "title": "密码错误登录", "result": "失败" }
      ]
    },
    {
      "sheet_name": "支付用例",
      "count": 30,
      "counts": { "通过": 15, "失败": 3, "阻塞": 0, "未执行": 12 },
      "testcases": [
        { "global_index": 25, "sheet_index": 0, "id": "TC-026", "title": "微信支付", "result": "通过" }
      ]
    }
  ]
}
```

每条用例只包含三个字段：`id`（序号）、`title`（用例名称）、`result`（测试结果）。若结果字段为空，显示为 `null`。

**API 3：`/api/sheet-meta/<sheet_name>` 返回格式**

```json
{
  "sheet_name": "测试备注",
  "headers": ["项目", "内容"],
  "rows": [
    ["测试环境", "QA环境"],
    ["测试版本", "v1.2.0"]
  ]
}
```

### 3.4 数据模型变更

`STATE` 全局状态扩展：

```python
STATE = {
    # 原有字段（保持兼容）
    'testcases': [],       # 所有用例 sheet 的用例合并后的列表（全局索引）
    'mapping': {},         # 主 sheet 的列映射（向后兼容）
    'headers': [],         # 主 sheet 的表头（向后兼容）
    'filepath': None,
    'filename': '',
    'sheet_name': None,    # 主 sheet（向后兼容）
    'header_row': 1,
    
    # 新增字段
    'sheets_info': [],     # 所有 sheet 的分类信息列表
    'testcase_sheets': [], # 测试用例 sheet 列表（含解析数据）
    'non_testcase_sheets': [], # 非测试用例 sheet 列表
}
```

每条用例字典新增字段：

```python
{
    # 原有字段 ...
    '_sheet_name': '登录用例',   # 来源 sheet 名称
    '_sheet_index': 0,           # 在该 sheet 内的索引（0-based）
}
```

**Sheet 用例数据缓存结构**（用于快速构建汇总）：

```python
STATE['testcase_sheets'] = [
    {
        'sheet_name': '登录用例',
        'header_row': 3,
        'mapping': {...},
        'headers': [...],
        'testcases': [{...}, {...}],  # 该 sheet 的用例列表
    },
    ...
]
```

### 3.5 读取流程改造

`read_testcases(filepath)` 改造为读取所有用例 sheet：

```
1. 打开 Excel，遍历所有 sheet
2. 对每个 sheet 调用 classify_sheet() 判断类型
3. 非用例 sheet → 存入 non_testcase_sheets（只存名称和原始数据）
4. 用例 sheet → 解析用例数据，存入 testcase_sheets
5. 将所有用例 sheet 的用例按 sheet 顺序合并为 testcases 列表
6. 每个用例标注 _sheet_name 和 _sheet_index
7. 返回完整数据包
```

### 3.6 保存结果适配

`save_result` 需要适配多 sheet：
- 从用例的 `_sheet_name` 字段获取目标 sheet
- 写入正确的 sheet 和正确的行号
- 若 `_sheet_name` 为空，回退到当前主 sheet（向后兼容）

## 五、边界条件

### 5.1 正常流程

1. 用户上传/加载 Excel 文件
2. 后端自动解析所有 sheet，分类用例/非用例
3. 用例 sheet 的用例合并到全局列表，非用例 sheet 单独存储
4. 前端初始化时加载所有数据
5. 用户切换到「汇总统计」视图，看到按 sheet 分组的用例列表
6. 用户点击某条用例名称，跳转回执行视图并定位到该用例

### 5.2 异常处理

| 异常情况 | 处理方式 |
|----------|----------|
| 单个 sheet 无数据行 | 显示为 0 条用例，列表显示「暂无测试用例」 |
| 用例缺少结果字段 | 测试结果列显示为空（不显示任何标签） |
| 整个文件无测试用例 sheet | 汇总统计页面显示「暂无测试用例」，隐藏分组列表 |
| 所有 sheet 均为非用例 | 同上，执行视图显示空状态 |
| 非用例 sheet 为空 | 元数据视图显示「该 sheet 无数据」 |
| 用例 sheet 无 id 列 | 序号列使用 sheet 内索引（#1, #2...） |
| 用例 sheet 无 title 列 | 名称列显示「(无标题)」 |
| 保存结果时 sheet 已被删除 | 返回错误提示，提醒用户刷新 |

### 5.3 向后兼容

- 单 sheet Excel 文件：行为与当前完全一致，不破坏任何功能
- 现有 API 返回值格式不变，新增字段不影响前端
- 保存结果逻辑兼容单 sheet（回退机制）

- [ ] 空 sheet 显示「暂无测试用例」
